Lossless data compression is a fundamental concept in computer science. 
It allows information to be reduced in size without losing any of the original content. 
When data is decompressed, it must be identical to the input. This requirement makes 
lossless compression essential for text documents, source code, configuration files, 
logs, and other structured data.

Two classical lossless compression techniques are Huffman coding and LZ78. 
Huffman coding is an entropy-based method that assigns shorter binary codes to more 
frequent symbols and longer codes to less frequent symbols. This approach works well 
when character frequencies are unevenly distributed. Natural language text typically 
contains such distributions, as spaces and common letters appear more frequently than 
rare characters.

LZ78 is a dictionary-based compression algorithm. Instead of assigning codes to 
individual characters, it builds a dictionary of repeated substrings encountered 
during processing. When a repeated pattern is found, the algorithm replaces it with 
a reference to a previously stored entry. This technique is effective for inputs 
containing recurring sequences, such as repeated words, structured patterns, or 
formatted data.

Compression performance depends heavily on the structure of the input data. 
For example, highly repetitive strings like “aaaaaa” compress extremely well, 
while random data is typically incompressible. Natural language lies somewhere 
in between: it contains structure, repetition, and predictable frequency patterns.

In practical applications, compression reduces storage requirements and network 
bandwidth usage. Smaller files can be transmitted faster and stored more efficiently. 
However, compression also introduces computational overhead during encoding and 
decoding. Therefore, algorithm choice depends on trade-offs between compression ratio, 
speed, and implementation complexity.

Modern compression systems often combine multiple techniques to achieve better 
performance. Even though Python is not optimized for high-performance compression, 
it provides a clear and readable environment for understanding algorithmic principles 
and analyzing compression behavior.

This project compares Huffman coding and LZ78 using natural-language inputs of 
increasing size. The goal is not only to verify correctness but also to observe 
how compression ratios evolve as file sizes grow from kilobytes to megabytes. 
Such empirical comparison provides insight into the strengths and weaknesses 
of each approach.
